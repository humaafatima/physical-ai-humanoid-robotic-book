# Glossary

Technical terms and acronyms used throughout this book.

## A

**AMCL** - Adaptive Monte Carlo Localization. A probabilistic localization algorithm for robots moving in 2D spaces.

**ADR** - Automatic Domain Randomization. Technique for automatically adjusting simulation parameters to improve sim-to-real transfer.

## D

**DDS** - Data Distribution Service. Middleware used by ROS 2 for inter-process communication.

**Digital Twin** - A virtual representation of a physical robot that accurately simulates its behavior.

## E

**Embodied AI** - Artificial intelligence systems that interact with the physical world through a robot body.

**EKF** - Extended Kalman Filter. Algorithm for sensor fusion and state estimation.

## I

**IMU** - Inertial Measurement Unit. Sensor combining accelerometer, gyroscope, and sometimes magnetometer.

**Isaac ROS** - NVIDIA's GPU-accelerated ROS 2 packages for robotics perception.

**Isaac Sim** - NVIDIA's robotics simulation platform built on Omniverse.

## L

**LLM** - Large Language Model. Neural networks trained on vast text data for natural language understanding.

## M

**MoveIt** - Motion planning framework for robotic manipulation in ROS.

## N

**Nav2** - ROS 2 navigation stack for autonomous mobile robot navigation.

## R

**ROS 2** - Robot Operating System 2. Middleware framework for robot software development.

**RGB-D** - Red-Green-Blue-Depth. Camera that captures color images and depth information.

## S

**SLAM** - Simultaneous Localization and Mapping. Algorithm for building a map while tracking robot position.

**Sim-to-Real** - Transfer of policies or behaviors learned in simulation to physical robots.

## U

**URDF** - Unified Robot Description Format. XML format for describing robot kinematics and dynamics.

**USD** - Universal Scene Description. File format used by NVIDIA Omniverse for 3D scenes.

## V

**VLA** - Vision-Language-Action. Models that combine visual perception, natural language understanding, and robotic control.
